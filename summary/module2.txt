# 📜 In-Depth Final Summary – Module 2: Price Flow Analyzer (Perfect Version)

**Purpose**: Module 2 is a standalone component of the BTC/USD prediction system, designed to detect significant price movements (≥300 points, including continuations like 2000+ points) in 1-minute Binance candles using only the 12-field data (`timestamp`, `open`, `high`, `low`, `close`, `volume`, `quote_volume`, `trades`, `taker_buy_base`, `taker_buy_quote`, `volatility`, `datetime_ist`). It generates **Buy**, **Sell**, or **Neutral** signals with confidence, Stop Loss (SL), Take Profit 1 (TP1), and Take Profit 2 (TP2), targeting ~85% accuracy. The module uses a hybrid approach combining LSTM with Attention for pattern recognition and Dynamic Time Warping (DTW) for matching current flows with historical patterns. It remains independent of whale detection (Module 3) and news sentiment (Module 4), with fusion occurring later in Module 5. Built with free tools (pandas, PyTorch, scikit-learn) and data sources (Binance API, Kaggle, CryptoDataDownload), it’s optimized for low-spec systems and real-time WebSocket integration.

---

## 🎯 Key Features
- **Input**: 50,000 historical 1-minute candles with 12 fields, processed into 100-candle sliding windows (step=1) for training.
- **Detection Methods**: Combines three approaches—Price Momentum, Volume/Activity Surge, Volatility/Taker Pressure—to identify 300+ point moves.
- **Pattern Matching**: Uses FastDTW to match current flow with past patterns, enhancing LSTM predictions.
- **Signal Logic**:
  - **Buy**: ≥300-point upward move without touching the candle’s `low` (with 50-point buffer).
  - **Sell**: ≥300-point downward move without touching the candle’s `high` (with 50-point buffer).
  - **Neutral**: No qualifying move (e.g., <300 points or touches both `high`/`low`).
- **Output**: JSON file (`price_flow_detections.json`) with signal, confidence, SL, TP1, TP2, and detailed metrics.
- **Session Adaptation**: Adjusts thresholds for Asia (00-08 UTC), London (08-16 UTC), NY (16-00 UTC), and Sundays.
- **Training**: LSTM trained on 50,000 candles, validated via backtesting; DTW matches historical sequences.
- **Resources**: Free tools (pandas, numpy, PyTorch, scikit-learn, backtrader) and data (Binance API/WebSocket, Kaggle, CryptoDataDownload).

## 📋 Detailed Implementation

### 1️⃣ Input Data & Preprocessing
- **Data**: 50,000 1-minute Binance candles with 12 fields:
  - `timestamp`: Integer (Unix ms, e.g., 1755256620000).
  - `open`, `high`, `low`, `close`: Floats (e.g., 119020.16, 119020.16, 118962.61, 118978.21).
  - `volume`, `quote_volume`, `taker_buy_base`, `taker_buy_quote`: Floats (e.g., 5.29589, 630097.67, 2.59083, 308227.63).
  - `trades`: Integer (e.g., 2549).
  - `volatility`: Float (rolling std dev of returns * 100 over 10 prices, e.g., 0.0).
  - `datetime_ist`: String (e.g., "2025-08-15 16:47:00").
- **Preprocessing**:
  - Normalize floats (0-1 scale) for `open`, `high`, `low`, `close`, `volume`, `quote_volume`, `taker_buy_base`, `taker_buy_quote`, `volatility`.
  - Convert `timestamp` to session (Asia: 00-08 UTC, London: 08-16 UTC, NY: 16-00 UTC) using `datetime_ist` and pytz.
  - Create 100-candle sliding windows with step=1 (e.g., candles 1-100, 2-101, 3-102), yielding ~49,900 sequences for training.
- **Output**: `core_sequences.pkl` (normalized sequences for LSTM and DTW).
- **Why Robust**: Step=1 windows ensure comprehensive pattern capture; normalization handles varying scales; session detection enables adaptive analysis.

### 2️⃣ Signal Detection Logic
Module 2 generates signals based on the following logic, with a 50-point buffer for minor retracements to handle crypto volatility:

#### Buy Logic
- **Condition**: After a candle, if the market moves ≥300 points upward (single or multiple candles) without breaking below the candle’s `low` + 50-point buffer, generate **Buy signal**.
  - Example: Candle `low`=118,900, next candles reach 119,200 (300+ points) without going below 118,950 → Buy.
- **TP1**: First stable range after move, defined as `volatility` < 0.5x 20-candle average and `high`-`low` < 100 points for 3 consecutive candles. Use average `close` of range.
- **TP2**: If market continues upward after TP1 and stabilizes again (same criteria), use average `close` of new range.
- **SL**: Set at 2x recent `volatility` below entry price (e.g., entry=119,000, `volatility`=0.1 → SL=118,800). If confidence >0.9, tighten to 1.5x `volatility`.
- **Confidence**: Float (0-1) from LSTM (60%) and DTW (40%), based on 12-field patterns (e.g., rising `close`, high `taker_buy_base`, increasing `volatility`).

#### Sell Logic
- **Condition**: After a candle, if the market moves ≥300 points downward without breaking above the candle’s `high` - 50-point buffer, generate **Sell signal**.
  - Example: Candle `high`=119,000, next candles reach 118,700 without going above 118,950 → Sell.
- **TP1**: First stable range (`volatility` < 0.5x avg, `high`-`low` < 100 points, 3 candles). Use average `close`.
- **TP2**: If market continues downward and stabilizes again, use average `close`.
- **SL**: 2x `volatility` above entry (e.g., entry=118,700, `volatility`=0.1 → SL=118,900). Tighten to 1.5x if confidence >0.9.
- **Confidence**: LSTM + DTW probability based on patterns (e.g., falling `close`, high sell pressure via `taker_buy_base`).

#### Neutral Logic
- **Condition**: If neither Buy nor Sell conditions are met (e.g., move <300 points or market touches both `high` and `low` within buffer), generate **Neutral signal**.
  - Example: Candle `high`=119,000, `low`=118,900, next candles move 220 points or touch both → Neutral.
- **Confidence**: LSTM + DTW probability for stable patterns (e.g., low `volatility`, small `close` changes).
- **SL/TP**: Not applicable; log `volatility` and range for context.

**Why Robust**: Symmetrical logic with 50-point buffer ensures practical detection in volatile markets. Precise stability rules (`volatility` <0.5x avg, range <100 points) make TP1/TP2 reliable. Confidence combines LSTM and DTW for accuracy.

### 3️⃣ Detection Methods
Three methods identify potential moves, feeding into the LSTM+DTW hybrid:
1. **Price Momentum**:
   - Calculate cumulative `close` change over 20 candles.
   - Threshold: ≥300 points (Asia: 250, London: 300, NY: 350, Sundays: 280).
   - Example: `close` from 118,900 to 119,250 = 350 points → Triggers Buy check.
2. **Volume/Activity Surge**:
   - Check `volume` or `trades` > 2x 20-candle average (1.8x in Asia/Sundays).
   - Example: `volume`=12 BTC, avg=5 BTC → Surge ratio=2.4.
3. **Volatility/Taker Pressure**:
   - Check `volatility` > 1.5x 20-candle average (1.3x in Asia) or `taker_buy_base`/`taker_buy_quote` > 60% of `volume`/`quote_volume`.
   - Example: `volatility`=0.12 (avg=0.08), `taker_buy_base`=70% → Buy pressure.

**Why Robust**: Multiple methods ensure no move is missed. Session-specific thresholds (data-driven via backtesting) align with market liquidity.

### 4️⃣ Pattern Matching with DTW
- **Purpose**: Matches current 100-candle sequence with historical patterns to predict moves, addressing the belief that “current flow matches past flows.”
- **Implementation**:
  - Use FastDTW (scikit-learn compatible) to compute similarity distance between current sequence (12 fields, normalized) and ~49,900 historical sequences.
  - Select top 5 matches; analyze outcomes (e.g., 80% led to 300+ point up move).
  - Combine DTW confidence (based on distance, 40% weight) with LSTM probability (60% weight).
- **Example**: Current flow (rising `close`, high `volume`) matches 2022 bull run pattern → Predicts up move with 0.85 confidence.
- **Optimization**: Sample recent 10,000 candles for faster DTW on low-spec systems.

**Why Robust**: DTW handles time stretches (e.g., slow vs. fast moves), boosting accuracy and explainability. FastDTW ensures efficiency.

### 5️⃣ Machine Learning Model
- **Model**: LSTM with Attention (PyTorch).
  - **Input**: 100-candle sequences (12 fields, normalized).
  - **Architecture**: 2 layers, 64 hidden units, Attention to prioritize key candles (e.g., high `volatility` or `volume` spikes).
  - **Output**: Probability for Buy/Sell/Neutral, estimated move size.
- **Training**:
  - Dataset: 50,000 candles (~49,900 sequences).
  - Labels: Buy (≥300-point up move, untouched `low` ±50), Sell (≥300 down, untouched `high` ±50), Neutral (else).
  - Hyperparameters: Learning rate=0.001, epochs=50, batch size=32 (low-spec friendly).
  - Early stopping if validation loss plateaus (5 epochs).
- **Inference**: Real-time WebSocket data (via `websocket_client.py`) fed into 100-candle rolling buffer; output predictions per candle.
- **Hybrid with DTW**: LSTM learns patterns, DTW matches historical flows, combined for final confidence.

**Why Robust**: LSTM captures temporal dependencies, Attention focuses on key candles, DTW adds historical context. Batch size and early stopping ensure efficiency.

### 6️⃣ Session Adaptations
- **Logic**: Adjust thresholds based on session (detected via `timestamp`/`datetime_ist`):
  - **Asia (00-08 UTC)**: Move=250 points, volume=1.8x avg, volatility=1.3x avg.
  - **London (08-16 UTC)**: Move=300, volume=2x, volatility=1.5x.
  - **NY (16-00 UTC)**: Move=350, volume=2.2x, volatility=1.7x.
  - **Sundays**: Reduce volume/activity thresholds by 20%.
- **Implementation**: Python conditionals, tuned via backtesting on historical session data.

**Why Robust**: Session-specific thresholds reflect crypto market dynamics, validated by data-driven tuning.
